# BERT-based Sentence Embedding Model Trainer

This directory contains the code to train a sentence embedding model using `sentence-transformers`. The training can be done in two modes:
1.  **Unsupervised (SimCSE):** Uses self-supervised contrastive learning. The input is a simple list of sentences.
2.  **Supervised (Triplet Loss):** Uses labeled data in the format of `(anchor, positive, negative)` triplets to fine-tune the model.

## Prerequisites

-   Docker
-   An input CSV file generated by one of the scripts in the `../gen_data` directory.

## Step 1: Generate Training Data

First, you need to generate the training data from your source CSV file. The data generation scripts are in the `../gen_data` directory.

**Example:**
Navigate to the root of the project.

```bash
# Example for generating unsupervised and supervised data
python gen_data/simcse_data_gen_basic.py \
    --data_path /path/to/your/source.csv \
    --output_unsupervised data/unsupervised_samples.csv \
    --output_supervised data/supervised_samples.csv
```
*Make sure the output directory (e.g., `data/`) exists or is mounted in the Docker container.*

## Step 2: Build the Docker Image

From the root directory of this project, build the Docker image:

```bash
docker build -t bert-trainer .
```

## Step 3: Run the Training

Use `docker run` to execute the training script. You need to mount a volume for your data and another for the output model.

-   `-v $(pwd)/data:/app/data`: Mounts a local `data` directory (containing your generated CSVs) into the container.
-   `-v $(pwd)/models:/app/models`: Mounts a local `models` directory where the trained model will be saved.

### Unsupervised Training Example

This command trains a model using the unsupervised (SimCSE) method.

```bash
docker run --gpus all -it --rm \
    -v $(pwd)/data:/app/data \
    -v $(pwd)/models:/app/models \
    bert-trainer \
    python bert_training/train.py \
        --mode unsupervised \
        --train_file /app/data/unsupervised_samples.csv \
        --model_name klue/bert-base \
        --output_dir /app/models/klue-bert-base-unsupervised \
        --epochs 1 \
        --batch_size 64 \
        --lr 3e-5
```

### Supervised Training Example

This command trains a model using the supervised (triplet) method. It assumes your CSV has `anchor`, `positive`, and one or more `negative_...` columns.

```bash
docker run --gpus all -it --rm \
    -v $(pwd)/data:/app/data \
    -v $(pwd)/models:/app/models \
    bert-trainer \
    python bert_training/train.py \
        --mode supervised \
        --train_file /app/data/supervised_samples.csv \
        --model_name klue/bert-base \
        --output_dir /app/models/klue-bert-base-supervised \
        --epochs 3 \
        --batch_size 32 \
        --lr 2e-5
```

### Notes

-   `--gpus all`: This flag enables the use of all available GPUs. If you don't have a GPU or NVIDIA's container toolkit, remove this flag and change the `--device` argument in the python command to `cpu`.
-   **Paths inside Docker:** Note that all paths for the script (`--train_file`, `--output_dir`) must be paths *inside* the container (e.g., `/app/data`, `/app/models`).
-   **Model Caching:** The first time you run training for a specific model, `sentence-transformers` will download the pre-trained weights. To avoid re-downloading, you can mount a cache volume: `-v $(pwd)/.cache:/root/.cache`.
